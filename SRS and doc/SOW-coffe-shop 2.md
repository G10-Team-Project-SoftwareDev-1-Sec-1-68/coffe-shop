## 1. วัตถุประสงค์การจัดทำ Statement of Work
เอกสารนี้อธิบายข้อกำหนดความต้องการซอฟต์แวร์สำหรับระบบจัดการร้านกาแฟที่รองรับการสั่งซื้อทั้งหน้าร้านและออนไลน์

## 2. วัตถุประสงค์โครงการ

| วัตถุประสงค์ | ตัวชี้วัดความสำเร็จ |
|------------|-------------------|
| 1. บริหารจัดการสต็อกวัตถุดิบ | - ลดเวลาในการจัดการสต็อก<br>- ลดข้อผิดพลาดของผู้ที่ทำการตรวจสอบวัตถุดิบ |
| 2. จัดการสมาชิกและแต้มสะสม | - ลูกค้าใช้งานระบบสมาชิกและ กลับมาซื้อสินค้าภายในร้าน<br>- เพิ่มยอดในการสั่งซื้อสินค้า |
| 3. ลดต้นทุนการดำเนินงาน | - ลดต้นทุนด้านการจ้างงานลง<br>- ลดค่าใช้จ่ายด้านเอกสาร อย่างน้อย 15 % |
| 4. ยกระดับความพึงพอใจของลูกค้า | - คะแนนความพึงพอใจ > 3.5/5<br>- อัตราการร้องเรียน < 5% |
| 5. ควบคุมการผลิตและคุณภาพเครื่องดื่ม | - พนักงานรับออเดอร์เครื่องดื่มได้เร็วขึ้น<br>- สามารถทำออเดอร์เครื่องดื่มสะดวกมากขึ้น |
| 6. จัดการระบบการชำระเงิน | - ลูกค้าสามารถชำระเงินภายในซอฟแวร์ได้<br>- เพิ่มความสะดวกสบายในการจ่ายเงิน |
| 7. วิเคราะห์ข้อมูลและออกรายงาน | - สามารถจัดการเอกสารสรุปยอดได้เร็วขึ้น<br>- ซอฟแวร์สามารถสรุปผลการขายรายเดือนได้|


## 3. ขอบเขตของโครงการ

### 3.1 ขอบเขตการดำเนินงาน

ระบบงานหลักประกอบด้วย:

| โมดูล | ฟังก์ชันการทำงาน | ผู้ใช้งาน |
|-------|-----------------|-----------|
| 1. ระบบหน้าร้าน (POS) | - รับออเดอร์และสามารถตรวจสอบ<br>- จัดการคำสั่งซื้อ<br>- ออกใบเสร็จ | - เจ้าของธุระกิจ<br>- พนักงานต้อนรับ |
| 2. แอปพลิเคชันสำหรับลูกค้า | - ชำระผ่านบัตรเครดิต/เดบิต<br>- โอนเงินผ่านธนาคาร<br>- E-Wallet | - ลูกค้า<br>- ผู้จัดการ |
| 3. ระบบจัดการสมาชิก | - ลูกค้าสามารถสมัครใช้งานได้<br>- สามารถตรวจสอบข้อฒุลของสมาชิกได้<br>- จัดการโปรโมชั่น | - ผู้จัดการ<br>- เจ้าของร้าน |
| 4. ระบบจัดการสต็อก | - ตรวจสอบสต็อก<br>- เตือนเมื่อวัตถุดิบใกล้หมด<br>- วิเคราะห์ว่าวัตถุดิบอันไหนหมดเร็ว | - ผู้จัดการ<br>- คนดูแลสต็อก |
| 5. ระบบจัดการพนักงาน | - บอกเวลาเข้างานของพลักงาน<br>- เก็บ log การทำงานในแต่วัน<br>- รายงานวิเคราะห์ความสามารถในการทำงานของพนักงาน | - เจ้าของร้าน<br>- ผู้จัดการ |
| 6. ระบบรายงานและวิเคราะห์ | - รายงานการสั่งยอดการสั่งของแต่ละเมนู<br>- รายงานรายได้<br>- รายงานวิเคราะห์ | - เจ้าของร้าน<br>- ผู้จัดการ |


### 3.2 ระยะเวลา
โครงการมีระยะเวลาดำเนินการทั้งสิ้น 6 เดือน ตั้งแต่วันที่ 1 มกราคม 2025 ถึง 30 มิถุนายน 2025

### 3.3 งวดงานและงวดเงิน

| งวดที่ | ระยะเวลา | ขอบเขตงาน | เงื่อนไขการส่งมอบ | สัดส่วนการจ่ายเงิน |
|-------|----------|------------|------------------|------------------|
| 1 | 2 เดือน | - วิเคราะห์และออกแบบระบบ<br>- จัดทำ Prototype | - เอกสาร SRS<br>- Prototype ที่ได้รับการอนุมัติ | 30% |
| 2 | 2 เดือน | - พัฒนาระบบจองและชำระเงิน<br>- ทดสอบระบบ | - ระบบผ่านการทดสอบ<br>- คู่มือการใช้งาน | 40% |
| 3 | 2 เดือน | - พัฒนาระบบจัดการและรายงาน<br>- ติดตั้งระบบ<br>- อบรมผู้ใช้งาน | - ระบบทำงานสมบูรณ์<br>- ผู้ใช้ผ่านการอบรม | 30% |

### 3.4 การรายงานความคืบหน้าโครงการ
การรายงานความคืบหน้าจะดำเนินการดังนี้:
1. รายงานประจำสัปดาห์: ส่งทุกวันศุกร์ เวลา 16:00 น.
2. ประชุมติดตามงานรายเดือน: จัดทุกวันจันทร์แรกของเดือน
3. รายงานสรุปประจำงวดงาน: ส่งเมื่อสิ้นสุดแต่ละงวด

### 3.5 ปัจจัยที่มีผลกระทบต่อโครงการ

| ปัจจัย | ผลกระทบ | มาตรการรองรับ |
|-------|----------|---------------|
| การเปลี่ยนแปลงความต้องการ | - ล่าช้า<br>- เพิ่มต้นทุน | - Change Control Process<br>- Buffer Timeline 15% |
| ความพร้อมของผู้ใช้งาน | - การใช้งานระบบ<br>- การยอมรับระบบ | - แผนการฝึกอบรม<br>- Change Management |
| ข้อจำกัดด้านเทคนิค | - ประสิทธิภาพ<br>- การเชื่อมต่อ | - POC ก่อนพัฒนา<br>- Performance Test |

## 5. การบริหารการเปลี่ยนแปลง
การบริหารการเปลี่ยนแปลงจะดำเนินการผ่าน Change Control Board (CCB) โดยมีขั้นตอนดังนี้:
1. การยื่นคำขอเปลี่ยนแปลง (Change Request)
2. การประเมินผลกระทบ
3. การอนุมัติการเปลี่ยนแปลง
4. การดำเนินการและติดตามผล
